# MAX17048 (IC3) — 1-Cell Li+ Fuel Gauge (ModelGauge)

Datasheet: Maxim/Analog MAX17048/MAX17049, Rev 7, 11/16 (`max.pdf`). This reference covers the **MAX17048** single-cell variant only.

## Overview
The MAX17048 is a tiny micropower (3µA hibernate, 23µA active) I2C fuel gauge for a single Li+ cell using the ModelGauge algorithm (no sense resistor, no learn cycle) (p.1). It is offered in a 0.9mm x 1.7mm 8-bump WLP or a 2mm x 2mm 8-pin TDFN package (p.1). On this board it monitors the single-cell battery state-of-charge and reports VCELL/SOC/rate to the STM32WB host over I2C; it is powered directly from the cell (VDD = battery+) and uses one external 0.1µF bypass cap (p.5, p.14). Note: the '48 has **no current-sense resistor**, so it reports SOC/voltage only, not coulomb-counted current.

## Power & supply
- **Supply voltage (VDD):** 2.5V to 4.5V (Note 2) (p.1). On the '48, VDD is tied to the battery and also serves as the voltage-measurement input; VDD measurement range 2.5–5V (p.1).
- **Absolute maximum ratings (p.1):**
  - CELL to GND: −0.3V to +12V
  - All other pins to GND: −0.3V to +6V
  - Data I/O pins (SCL, SDA, ALRT) operating range: −0.3V to +5.5V (Electrical Characteristics, p.1)
  - Continuous sink current, SDA/ALRT: 20mA
  - Operating temperature: −40°C to +85°C; storage −55°C to +125°C
- **Decoupling:** bypass VDD with **0.1µF to GND** — this is the "only one external component" in the simple circuit (p.0, p.5, p.14).
- **Sequencing:** no explicit sequencing requirement stated; because VDD = battery, the part powers up with the cell. POR (and battery insertion) triggers a quick-start, so VDD should only see a relaxed battery for a good initial SOC (p.9). Bus-low for >2.5s puts the IC into shutdown/sleep (Note 6, p.2).

## Configuration / strap & mode pins
The MAX17048 has **no interface-select, no I2C-address-select, and no output-voltage/charge-current pins** — it is a fixed-address I2C-only fuel gauge. The only hardware-config pins are CTG and QSTRT.

**Table 3 — Possible Application Configurations (MAX17048 rows only) (p.14):**

| System configuration | VDD | ALRT | QSTRT |
|----------------------|-----|------|-------|
| 1S pack-side location | Power directly from battery | Leave unconnected | Connect to GND |
| 1S host-side location | Power directly from battery | Leave unconnected | Connect to GND |
| 1S host-side, low-cell interrupt | Power directly from battery | Connect to system interrupt | Connect to GND |
| 1S host-side, hardware quick-start | Power directly from battery | Leave unconnected | Connect to rising-edge reset signal |

- **CTG:** always "Connect to Ground" (p.5). No mode-select function.
- **QSTRT:** tie to GND for normal operation; only route to a host GPIO if hardware quick-start is intended (rising edge = quick-start) (p.5, p.9, p.14). Quick-start is **not recommended for most systems** (the IC handles startup transparently) (p.9).

## Interface
- **Bus:** I2C, slave-only, up to **400kHz** (fSCL 0–400kHz, Note 7) (p.2, p.13). No special config to run at any speed up to 400kHz (p.16).
- **Address (FIXED):** 7-bit slave address is **0x36** — datasheet expresses it as **0x6C (write) / 0x6D (read)** 8-bit, binary `0110110` (p.15). There is **no address-select pin**; address is fixed (p.15).
- **Registers:** 16-bit words only; 8-bit writes have no effect; multibyte values are MSB-first, MSB on even addresses (p.11, p.15). Key registers: VCELL 0x02, SOC 0x04, MODE 0x06, VERSION 0x08, HIBRT 0x0A, CONFIG 0x0C, VALRT 0x14, CRATE 0x16, VRESET/ID 0x18, STATUS 0x1A, CMD 0xFE (p.11–12). POR/reset command = write 0x5400 to CMD (0xFE); device does NOT ACK that command (p.13).
- **Disconnect sensing:** SCL and SDA have internal pulldowns (IPD ≈ 0.2–0.4µA) so the IC can detect a disconnected/low bus; holding both low > ~2.5s forces shutdown (Note 5/6, p.2, p.5).
- No SPI, no 3-/4-wire selection (I2C only). No power-up protocol-detection step.

## Application circuit / external components
From the simple circuit (p.0), Figure 14 (p.14), and the pin table (p.5):
- **0.1µF** ceramic bypass cap from **VDD to GND** — the only required external component (p.0, p.5, p.14).
- **I2C pull-up resistors** on SDA and SCL — "the system must provide pullup circuits for ALRT (if used), SDA, and SDL [SCL]" (p.13). **No resistor values given in the datasheet** — not stated; size per I2C bus (≤400pF bus cap, p.2).
- **ALRT pull-up** if ALRT is used (open-drain to host interrupt) (p.5, p.13). Value not stated in datasheet.
- **QSTRT → GND** (default) (p.5, p.14).
- **CTG → GND**, **EP → GND** (p.5).
- No inductor, no diode, no sense resistor required (ModelGauge eliminates the sense resistor) (p.1).

## Key electrical specs
All from Electrical Characteristics unless noted (p.1–2):
- **Supply current:** Active mode (IDD1) 23µA typ / 40µA max; Hibernate 3µA typ (reset comparator disabled) / 4µA (enabled); Sleep 0.5µA typ / 2µA max (p.1).
- **Logic-in thresholds (SDA, SCL, QSTRT):** VIH ≥ 1.4V min; VIL ≤ 0.5V max (p.1).
- **Output low (SDA, ALRT):** VOL ≤ 0.4V at IOL = 4mA (p.1).
- **Bus low-detection current (SDA/SCL internal pulldown):** IPD 0.2–0.4µA at 0.4V (always present) (p.1–2).
- **Bus low-detection / shutdown timeout:** tSLEEP 1.75–2.5s (p.1); IC enters shutdown after SCL & SDA below VIL for >2.5s (Note 6, p.2).
- **Voltage measurement:** resolution 1.25mV/cell; range (VDD pin, '48) 2.5–5V; accuracy ±7.5mV/cell typ at VCELL=3.6V/25°C (±20mV over range) (p.1).
- **VRESET (SOC reset / battery-swap) range:** configurable 2.28V–3.48V in 40mV steps; trimmed 3.0V default (p.1, p.13).
- **SCL clock:** 0–400kHz (p.2).
- **Logic pins** tolerate up to 5.5V (operating) — safe with 3.3V host (p.1).

## Schematic-review checklist
- [ ] **VDD = battery+**, not 3.3V rail. On the '48, VDD is both supply and the cell voltage-sense input — it must connect to BAT+ so SOC is measured (p.5). If powered from a regulated 3.3V instead of the cell, the gauge reads the regulator, not the battery (that wiring is the MAX17049 mode).
- [ ] **0.1µF from VDD to GND** present and close to the pin (p.5).
- [ ] **CTG (pin 1) tied to GND** — it has no other function (p.5). Confirm it is not left floating or mis-net.
- [ ] **QSTRT (pin 6) tied to GND** unless hardware quick-start is genuinely wanted; floating QSTRT risks spurious resets (p.5, p.14).
- [ ] **EP (exposed pad) tied to GND** on TDFN package (p.5).
- [ ] **SDA and SCL pull-ups present** (chip does not provide them; only weak ~0.2–0.4µA pulldowns for disconnect sensing). Confirm pull-up to the host I2C rail (≤3.3V) (p.5, p.13).
- [ ] **ALRT pull-up** if ALRT routes to the STM32WB interrupt — it is open-drain; without a pull-up the interrupt never goes high. If ALRT is unused, leave it unconnected (p.5, p.14).
- [ ] **I2C address is FIXED at 0x36 (8-bit 0x6C/0x6D)** — there is no select pin. Check for address conflict on the shared I2C bus with the other sensors (BMA400, BMP280/BME280). 0x36 is distinct from typical BMx addresses, but verify no overlap (p.15).
- [ ] **CELL pin (pin 2)**: on the '48 it is internally not connected; datasheet says connect to BAT+. Either tie to BAT+ or leave as the layout requires — do not assume it carries the sense function (that's VDD on the '48) (p.5).
- [ ] Confirm logic levels: I2C pins must not exceed 5.5V abs-max; CELL pin abs-max is +12V, all other pins +6V (p.1).
- [ ] Bus must not be held low >2.5s in normal operation or the IC drops to shutdown/sleep (Note 6, p.2).
