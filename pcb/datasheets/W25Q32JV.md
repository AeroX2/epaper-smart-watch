# W25Q32JV — 32M-bit (4 MByte) 3V Serial NOR Flash (IC2)

Datasheet: Winbond W25Q32JV, Revision G, March 27 2018 (`w25.pdf`). Page citations refer to the printed page footer numbers, which match the dump tool's `PAGE N` index for this file.

## Overview
The W25Q32JV is a 32M-bit (4 MByte) 3V serial NOR flash supporting Standard/Dual/Quad SPI, organized as 16,384 pages of 256 bytes, with 4 KB sectors and 64 KB blocks (p.4). On this board it is IC2, the off-chip data/code/asset flash for the STM32WB5MMG, hung on an SPI bus (CLK, /CS, DI/IO0, DO/IO1, plus /WP, /HOLD). The intended package is the 8-pad WSON 6x5 / 8x6 mm (or the pin-compatible 8-pin SOIC/VSOP), all of which share the same 8-pad pinout (p.5). Supply is a single 2.7–3.6V rail (p.4, p.59).

## Pinout
8-pad / 8-pin pinout — applies to SOIC 150/208-mil, VSOP 208-mil, WSON 6x5/8x6-mm, XSON 4x4-mm, and PDIP (p.5, p.8). The WSON also has a center exposed pad.

| Pin # | Name | Type | Function | Connect-to / notes |
|------|------|------|----------|--------------------|
| 1 | /CS | digital-IO (input) | Chip Select, active low (p.5) | Driven by host GPIO/SPI NSS. Pull-up to VCC recommended so /CS tracks VCC at power-up/down and so it idles high (p.9) |
| 2 | DO (IO1) | digital-IO | Data Output in Standard/Dual; bidirectional IO1 in Dual/Quad (p.5) | SPI MISO to host. IO1 in Dual/Quad modes (notes 1-2, p.5) |
| 3 | /WP (IO2) | digital-IO | Write Protect input (active low) in Standard/Dual; becomes IO2 when QE=1 (p.5, p.9) | For standard SPI use, tie to VCC (with pull-up) to disable HW status-register protection, or drive from host; must NOT float (see checklist). Becomes data line IO2 in Quad mode |
| 4 | GND | ground | Ground (p.5) | Ground plane |
| 5 | DI (IO0) | digital-IO | Data Input in Standard/Dual; bidirectional IO0 in Dual/Quad (p.5) | SPI MOSI from host. IO0 in Dual/Quad modes |

| 7 | /HOLD or /RESET (IO3) | digital-IO | Hold (active low) or Reset input in Standard/Dual; becomes IO3 when QE=1 (p.5, p.9) | For standard SPI, tie high (pull-up to VCC) or drive from host so /HOLD is inactive; must NOT float. ## Power & supply
- **Operating supply (VCC):** 3.0–3.6V when running at up to 133 MHz (FR=133MHz); 2.7–3.0V when limited to 104 MHz (FR=104MHz). Read-Data (03h) instruction is capped at fR=50MHz across the whole range (p.59, p.63).
- **Absolute maximum ratings (p.59):**
  - VCC: −0.6 to +4.6 V
  - Voltage on any pin (VIO): −0.6 to VCC+0.4 V (relative to GND)
  - Transient on any pin (<20ns): −2.0 V to VCC+2.0 V
  - Storage temperature: −65 to +150 °C
  - ESD (HBM): −2000 to +2000 V
- **Decoupling cap:** Not stated in datasheet (no recommended value/figure given). Standard practice is a 0.1 µF cap from VCC (pin 8) to GND (pin 4), but the value is not specified in this PDF.
- **Sequencing / power-up (p.9, p.60):**
  - /CS must track the VCC supply level during VCC ramp-up and ramp-down (a pull-up on /CS helps accomplish this).
  - tVSL: VCC(min) to /CS low ≥ 20 µs.
  - tPUW: delay before a write instruction ≥ 5 ms after power-up.
  - VWI (write-inhibit threshold): 1.0–2.0 V — below this, program/erase/write are ignored.
  - After power-up, /CS must transition high→low before a new instruction is accepted (p.9).

## Configuration / strap & mode pins
This part has **no hardware interface-select or address-select straps** — interface mode (Standard / Dual / Quad) is selected by SPI instruction opcodes and the non-volatile **Quad Enable (QE)** bit in Status Register-2 (S9), not by a pin (p.9, p.16). The only "hardware config" pins are /WP and /HOLD, whose behavior is set by the QE bit:

**QE bit (Status Register-2, S9) — controls IO2/IO3 vs /WP & /HOLD (p.16):**

| QE | /WP pin (pad 3) | /HOLD pin (pad 7) | Modes available | Factory default for |
|----|------------------|--------------------|------------------|----------------------|
| 0 | Write-Protect input | /HOLD input | Standard + Dual SPI | ordering options "IM" / "JM" |
| 1 | becomes IO2 (data) | becomes IO3 (data) | Standard + Dual + Quad SPI | ordering options "IQ" / "JQ" |

Important: the QE power-on default is **part-number dependent** — it is 0 (Standard/Dual, /WP & /HOLD active) for the -IM/-JM variants and 1 (Quad, /WP & /HOLD repurposed as IO2/IO3) for the -IQ/-JQ variants (p.16). Confirm which ordering suffix is actually placed (see checklist). The JEDEC/device IDs also differ: -IQ/JQ report 9Fh = EFh 4016h; -IM/JM report 9Fh = EFh 7016h (p.21).

**Output Driver Strength (Status Register-3, DRV1:DRV0) — software-set, default 25% (p.17):**

| DRV1, DRV0 | Driver strength |
|------------|------------------|
| 0,0 | 100% |
| 0,1 | 75% |
| 1,0 | 50% |
| 1,1 | 25% (default) |

## Interface
- **Bus:** SPI — supports Standard SPI, Dual SPI, and Quad SPI (Quad requires QE=1) (p.4, p.9).
- **SPI mode:** Mode 0 (CPOL=0, CPHA=0) and Mode 3 (CPOL=1, CPHA=1) supported. Mode 0 → CLK idles low at the /CS edges; Mode 3 → CLK idles high (p.11). Data clocked into DI is sampled on the rising edge of CLK, MSB first (p.9, p.21).
- **Max clock:** 133 MHz (all instructions except Read-Data 03h) at 3.0–3.6V; 104 MHz at 2.7–3.0V. Read-Data (03h) is limited to 50 MHz regardless of voltage (p.63). Equivalent throughput up to 266 MHz (Dual I/O) / 532 MHz (Quad I/O) (p.4).
- **3- vs 4-wire:** Standard SPI uses 4 wires (CLK, /CS, DI, DO) plus /WP and /HOLD as control. There is no 3-wire (half-duplex single-data) select pin; Dual/Quad reuse the DI/DO/WP/HOLD pads as IO0–IO3 (p.4, p.9).
- **Power-up protocol:** After power-up /CS must go high→low before the first instruction; no auto interface detection beyond the QE-bit-controlled pin mux (p.9, p.16).
- **No I2C** — this is an SPI-only device, so the I2C-address discussion in the template is N/A.

## Application circuit / external components
The datasheet does not include a full reference-design schematic with component values. Required/recommended externals stated in the text:
- **/CS pull-up resistor to VCC** — recommended so /CS tracks VCC at power-up/down and idles high (p.9). Value not stated; choose a moderate value (e.g. tens-to-hundreds kΩ) per host drive — *value not stated in datasheet*.
- **/WP (pad 3) handling for standard SPI** — must be tied high (to VCC, typically via pull-up) or driven; otherwise it floats. (Tying high disables the hardware status-register write-protect; pulling it controllable lets firmware use HW protection.) Active-low function described p.9.
- **/HOLD (pad 7) handling for standard SPI** — must be tied high (to VCC, typically via pull-up) or driven so the device is not inadvertently paused; /HOLD is active low (p.9).
- **VCC decoupling cap** — recommended in practice (0.1 µF typical), but *value not stated in datasheet*.
- No external inductors, diodes, sense resistors, or bulk caps are required (pure logic device).

- **Logic thresholds:** VIL = −0.5 V to VCC×0.3; VIH = VCC×0.7 to VCC+0.4 (p.61).
- **Output levels:** VOL ≤ 0.2 V @ IOL=100µA; VOH ≥ VCC−0.2 V @ IOH=−100µA (p.61).

- **Power-down current (ICC2):** typ 1 µA, max 15 µA (p.61).
- **Read current (ICC3):** typ 8 / max 15 mA @ 50 MHz; typ 10 / max 18 mA @ 80 MHz; typ 12 / max 20 mA @ 104 MHz (p.61).
- **Program/erase/write current (ICC4–ICC7):** typ 20 mA, max 25 mA (p.61).
- **Input/output capacitance:** CIN ≤ 6 pF, COUT ≤ 8 pF; leakage ±2 µA (p.61).

## Schematic-review checklist
- **/WP (pad 3) must not float** in standard SPI: tie high to VCC (pull-up) to leave HW write-protect disabled, or route to a controllable line. A floating /WP can leave status-register write-protect in an undefined state (p.9). Verify a net/pull-up exists.
- **/HOLD (pad 7) must not float** in standard SPI: tie high to VCC (pull-up) or drive it, otherwise the device can be inadvertently held/paused (active-low, p.9). Verify a net/pull-up exists.
- **/CS pull-up to VCC** present so /CS tracks VCC during power ramp and idles high after power-up (p.9, p.60). Verify resistor exists.
- **VCC decoupling cap** present at pin 8 to GND (value not specified by datasheet; use 0.1 µF). Confirm placement close to the pad.
- **Ordering-suffix vs intended mode:** if firmware uses **standard/dual SPI and relies on /WP & /HOLD as control pins**, the part must be a **QE-default-0 (-IM/-JM)** variant — a -IQ/-JQ part powers up with QE=1, repurposing pads 3 & 7 as IO2/IO3, which would make any /WP / /HOLD straps drive data lines. If using Quad mode, the opposite applies. Confirm the BOM part number matches the schematic's pin usage (p.16, p.21).
- **No hardware /RESET on this package:** the 8-pad part has no dedicated /RESET pin; pad 7's reset function is only the shared /HOLD-or-/RESET multiplexed input (HW /RESET pin exists only on SOIC-16 / TFBGA). Do not expect a separate reset line (p.9).
- **VCC rail within range for target clock:** 133 MHz needs 3.0–3.6V; at 2.7–3.0V the max clock drops to 104 MHz, and Read-Data 03h is always ≤50 MHz. Check the SPI clock the MCU drives vs the actual VCC rail (p.59, p.63).
- **Absolute-max on IO:** no pin may exceed VCC+0.4V (DC) — ensure the MCU SPI I/O voltage matches the flash VCC rail; do not drive 3.3V logic into a flash powered at a lower VCC (p.59).
- **EP (WSON exposed pad):** datasheet gives no electrical connection requirement; decide tie-to-GND vs float per layout convention — not specified in PDF.
