# BMA400 (AC1) — Bosch 3-axis Accelerometer

Datasheet: BST-BMA400-DS000-14, Rev 2.3 (07/2025), 121 pages.
PDF: `pcb/bst-bma400-ds000.pdf`

## Overview
The BMA400 is a 12-bit digital triaxial acceleration sensor with on-chip FIFO (1 KB), step counter,
activity/tap/orientation interrupt engine, and ultra-low power consumption (<14.5 µA in highest
performance, 160 nA sleep) (p.1, p.9). Package is a 12-pin LGA, 2 mm x 2 mm x 0.95 mm (p.1). It
supports I²C and SPI (3-/4-wire) plus 2 interrupt pins (p.1). On this smartwatch it is the motion
sensor (step counting, tap/wake gestures) on the shared digital bus; intended for exactly this kind
of "(smart) watch" / fitness application (p.1).

## Power & supply
- **VDD** (internal domains): min 1.72 V, typ 1.8 V, max 3.6 V (p.9). Pin table states 1.62 V…3.6 V (p.108).
- **VDDIO** (I/O domain): min 1.2 V, typ 1.8 V, max 3.6 V (p.9).
- VDD and VDDIO are independent; either can be 0 V while the other is in range. **If VDDIO is off, all
  interface pins (CSB, SDX, SCX) must be kept near GNDIO** (p.16).
- Power-on-reset only fires when BOTH VDD and VDDIO are established, so there's no protocol-detect
  risk from supply sequencing (p.101).
- **Absolute maximum ratings** (p.11):
  | Parameter | Min | Max | Units |
  |-----------|-----|-----|-------|
  | Voltage at VDD pin | -0.3 | 4 | V |
  | Voltage at VDDIO pin | -0.3 | 4 | V |
  | Voltage at any logic pin | -0.3 | VDDIO+0.3 (<4) | V |
  | ESD HBM (any pin) | | 2 | kV |
  | ESD CDM | | 500 | V |
  | Storage temp | -50 | +150 | °C |
- **Decoupling: 100 nF at pin 3 (VDDIO) and 100 nF at pin 7 (VDD)** — recommended in all three
  connection diagrams (p.108, p.109).
- Power-up time to sleep mode: typ 1 ms (p.9).

## Configuration / strap & mode pins
Two hardware-strap decisions: (1) interface protocol via CSB behavior at power-up, (2) I²C address via
SDO level.

**Protocol selection (CSB)** — auto-selected from CSB after power-up (p.101):
| CSB at/after power-up | Resulting interface |
|------------------------|---------------------|
| Tied to VDDIO during power-up, never toggled | I²C (default) — recommended to **hard-wire CSB to VDDIO** |
| Sees a rising edge after power-up | Switches to SPI until next reset/power-up (permanent for that session) |

Default mode at reset/power-up is I²C (p.100). A CSB rising edge is required before SPI use; for SPI
you must do a dummy single read (e.g. CHIP_ID) first — value is invalid — to trigger the switch (p.101).

**I²C address select (SDO)** — default address is `b001010X` (p.105):
| SDO pin | X | 7-bit I²C address |
|---------|---|-------------------|
| GND     | 0 | 0b0010100 = **0x14** (p.105, p.107) |
| VDDIO   | 1 | 0b0010101 = **0x15** (p.105) |

In I²C the SDO level **must be defined (tied to GND or VDDIO), not floating** (p.105).

## Interface
- **I²C** (default): SCL = SCX (pin 12), SDA = SDX (pin 2). Both lines need external pull-ups to VDDIO
  (p.105). 7-bit addressing only; supports standard, fast, and extended fast mode up to **3.4 MHz**
  (fSCL max 3400 kHz) (p.105). 7-bit address 0x14/0x15 set by SDO (see above). I²C bus load
  capacitance max 400 pF (p.100).
- **SPI**: 4-wire (default SPI) or software-selectable 3-wire mode (p.100). Pins: SCK = SCX, CSB = CSB,
  SDI/SDO on SDX/SDO (4-wire) or shared SDA on SDX (3-wire) (p.108). Max clock **fSPI = 17 MHz at
  VDDIO ≥ 1.71 V, 7 MHz at VDDIO < 1.71 V** (p.101). CPOL/CPHA mode not explicitly named as a number
  in the extracted text — SCK low/high pulse 20 ns each, SDI setup/hold 20 ns, CSB setup 50 ns for
  first transaction after the I²C→SPI switch then 20 ns, CSB hold 20 ns (p.101). (SPI mode 0
  / mode 3 designation: not stated in datasheet in the pages read.)
- **Power-up protocol detection**: I²C until a CSB rising edge switches it to SPI for the session
  (p.101). Internal ~120 kΩ pull-up on CSB to VDDIO means CSB idles high if left open (p.100).
- CHIP_ID register (0x00) reads **0x90** (p.47) — useful as a bring-up sanity check.

## Application circuit / external components
From the reference connection diagrams (p.108–110):
- **100 nF decoupling cap** on VDDIO (pin 3) to GND.
- **100 nF decoupling cap** on VDD (pin 7) to GND.
- **I²C mode**: pull-up resistors on SDA (SDX) and SCL (SCX) to VDDIO (value not specified in datasheet
  — size for ≤400 pF bus load and chosen speed). SDO strapped to GND (X=0) or VDDIO (X=1); CSB tied to
  VDDIO (p.110, p.108, p.105).
- **SPI mode**: no pull-ups shown; SDO/SDX/SCX/CSB to the SPI master; in 3-wire mode SDO is DNC (p.108–109).
- INT1/INT2 routed to host GPIOs or left DNC if unused (p.108).
- No inductors, diodes, or sense resistors required (digital sensor) — N/A.

## Key electrical specs
- **VIL** = max 0.3 × VDDIO; **VIH** = min 0.7 × VDDIO (SPI & I²C) (p.9).
- **VOL** (SPI, IOL=3 mA): ≤0.2 × VDDIO at VDDIO=1.8 V; ≤0.23 × VDDIO at VDDIO=1.2 V (p.9).
- **VOH** (SPI, IOH=3 mA): ≥0.8 × VDDIO at VDDIO=1.8 V; ≥0.62 × VDDIO at VDDIO=1.2 V (p.9).
- **CSB internal pull-up** to VDDIO: ~120 kΩ (p.100).
- Currents (nominal VDD/VDDIO, 25 °C): normal mode OSR=3 typ **14.5 µA**, OSR=0 typ 3.5 µA; sleep
  **160 nA**; low-power (25 Hz, OSR=0) typ 850 nA (p.9).
- Max I²C clock 3.4 MHz; max SPI clock 17 MHz (≥1.71 V) / 7 MHz (<1.71 V) (p.101, p.105).
- Operating temperature -40 °C…+85 °C (p.9).

## Schematic-review checklist
- [ ] **CSB (pin 10): for I²C, must be hard-wired to VDDIO.** If routed as a togglable line, any rising
      edge after power-up permanently switches the part to SPI for that session (p.101). Confirm it's
      strapped to VDDIO, not GND and not floating (internal ~120 kΩ pull-up to VDDIO exists but the
      datasheet says hard-wire it) (p.100, p.101).
- [ ] **SDO (pin 1): must be tied to GND or VDDIO in I²C** (not floating) — sets address 0x14 (GND) or
      0x15 (VDDIO) (p.105). Verify the chosen strap matches firmware's expected address and that 0x14/0x15
      does not collide with any other device on the same I²C bus (BMP280/BME280 default 0x76/0x77,
      MAX17048 0x36 — generally clear, but confirm) (p.105/p.107).
- [ ] **SDA (SDX/pin 2) and SCL (SCX/pin 12): need external pull-ups to VDDIO** for I²C — verify they
      exist (shared bus pull-ups count) and bus load ≤400 pF (p.105, p.100).
- [ ] **Decoupling: 100 nF on VDD (pin 7) and 100 nF on VDDIO (pin 3)** — both must be present (p.108).
- [ ] **VDD ≥ 1.72 V** per §1 spec (pin table says 1.62 V but spec table mandates 1.72 V min) — confirm
      the supplying rail is in range (p.9 vs p.108).
- [ ] **VDDIO must match the host logic rail** (1.2–3.6 V); thresholds are ratiometric (VIH=0.7×VDDIO,
      VIL=0.3×VDDIO) so the host I/O and pull-up rail must equal VDDIO (p.9).
- [ ] **GND (pin 9) and GNDIO (pin 8) both tied to ground** (p.108).
- [ ] **INT1 (pin 5)/INT2 (pin 6): if unused, leave Do-Not-Connect** — do not tie to a rail (p.99, p.108).
      If used, route to host GPIO; check polarity/push-pull vs open-drain in firmware (INT12_IO_CTRL).
- [ ] **Pins 4 and 11 are NC** — leave unconnected (p.108).
- [ ] **If VDDIO can be powered down independently of VDD**, ensure CSB/SDX/SCX are not driven high while
      VDDIO is off (must stay near GNDIO) (p.16).
- [ ] Confirm absolute-max not exceeded: no pin above VDDIO+0.3 V or above 4 V (p.11).
